RetinaNet 涉及的组件包括： Backbone、Neck、Head、BBox Assigner、BBox Encoder Decoder、Loss 和 BBox PostProcess。下面按照顺序结合代码详细分析。

Backbone

其中对应的backbone的内容

RetinaNet 算法采用了 ResNet50 作为 Backbone, 并且考虑到整个目标检测网络比较大，前面部分网络没有进行训练，BN 也不会进行参数更新。需要说明的是上述默认配置是经过前人工作和 OpenMMLab 在 COCO 数据集上不断实践的结果。推荐大家直接使用该配置模式，效果相对比较稳定。

对应的路径为 mmdet/models/backbones/resnet.py

(1) out_indices

ResNet 提出了骨架网络设计范式即 stem+n stage+ cls head，对于 ResNet 而言，其实际 forward 流程是 stem -> 4 个 stage -> 分类 head，stem 的输出 stride 是 4，而 4 个 stage 的输出 stride 是 4,8,16,32，这 4 个输出就对应 out_indices 索引。例如如果你想要输出 stride=4 的特征图，那么你可以设置 out_indices=(0,)，如果你想要输出 stride=4 和 8 的特征图，那么你可以设置 out_indices=(0, 1)。

stem（前置卷积层）：通常是 7x7 conv + BN + ReLU + MaxPool，用于快速降采样。

输出 stride = 4（输入图像尺寸缩小到 1/4）。

4 stages（阶段）

每个 stage 由多个 Bottleneck 或 BasicBlock 组成，逐步降采样。

各 stage 的输出 stride 分别为：

  • stage0 → stride=4（与 stem 相同）
  • stage1 → stride=8
  • stage2 → stride=16
  • stage3 → stride=32

这些 stride 值对应特征图相对于输入图像的缩放比例（如 stride=32 表示特征图尺寸是输入的 1/32）。

cls head（分类头）：用于分类任务（如 GlobalAvgPool + FC）

out_indices 指定需要保留哪些 stage 的输出，用于后续任务（如 FPN）。

  • 示例：
    • out_indices=(0,) → 只保留 stage0 的输出（stride=4）。
    • out_indices=(0, 1) → 保留 stage0 和 stage1 的输出（stride=4 和 8）。
    • out_indices=(0, 1, 2, 3) → 保留所有 4 个 stage 的输出（RetinaNet + FPN 的典型配置）。

因为 RetinaNet 后面需要接 FPN，故需要输出 4 个尺度特征图，简要代码如下：

  • res_layers：存储 ResNet 的 4 个 stage 的名称（如 ['layer1', 'layer2', 'layer3', 'layer4']）。
  • outs：最终输出的特征图列表，其内容由 out_indices 决定。

Bottleneck 和 BasicBlock 是两种不同的**残差块（Residual Block）**设计，用于构建网络的不同深度版本

(2) frozen_stages

该参数表示你想冻结前几个 stages 的权重，ResNet 结构包括 stem+4 stage

  • frozen_stages=-1，表示全部可学习
  • frozen_stage=0，表示stem权重固定
  • frozen_stages=1，表示 stem 和第一个 stage 权重固定
  • frozen_stages=2，表示 stem 和前两个 stage 权重固定

冻结权重需要同时满足两个条件：

  1. 设置为评估模式（eval）：关闭 Dropout、固定 BatchNorm 的均值和方差。
  2. 禁用梯度计算（requires_grad=False）：避免优化器更新参数。
  1. eval() 模式
    • 让 BatchNorm（BN）层使用预计算的全局均值/方差，而不是根据当前 batch 计算。
    • 关闭 Dropout 等训练时特有的行为。
  2. requires_grad=False
    • 告诉 PyTorch 不要计算这些参数的梯度，优化器不会更新它们。
  3. deep_stem
    • ResNet 的变种（如 ResNetV1d）用更复杂的 stem 结构（多个卷积层），普通 ResNet 只有 conv1 + bn1。

关键问题：为什么需要重写 train() 方法？

默认情况下，调用 model.train() 会将所有模块（包括已冻结的部分）切换回训练模式，导致 BN 层重新计算统计量。因此需要 在训练模式下强制保持冻结部分的 eval 状态。

(3) norm_cfg 和 norm_eval

norm_cfg 表示所采用的归一化算子，一般是 BN 或者 GN，而 requires_grad 表示该算子是否需要梯度，也就是是否进行参数更新，而布尔参数 norm_eval 是用于控制整个骨架网络的归一化算子是否需要变成 eval 模式。

RetinaNet 中用法是 norm_cfg=dict(type='BN', requires_grad=True)，表示通过 Registry 模式实例化 BN 类，并且设置为参数可学习。在 MMDetection 中会常看到通过字典配置方式来实例化某个类的做法， 底层是采用了装饰器模式进行构建，最大好处是扩展性极强，类和类之间的耦合度降低。

(4) style

style='caffe' 和 style='pytorch' 的差别就在 Bottleneck 模块中

Bottleneck 是标准的 1x1-3x3-1x1 结构，考虑 stride=2 下采样的场景，caffe 模式下，stride 参数放置在第一个 1x1 卷积上，而 Pyorch 模式下，stride 放在第二个 3x3 卷积上

torchvision 的 resnet 和早期 release 的 resnet 版本不一样，使得目标检测框架在使用 Backbone 的时候有两种不同的配置，不过目前新网络都是采用 PyTorch 模式。

Neck

Neck 模块即为 FPN，其简要结构如下所示：

前面说过 ResNet 输出 4 个不同尺度特征图 (c2,c3,c4,c5)，stride 分别是 (4,8,16,32)，通道数为 (256,512,1024,2048),通过配置文件我们可以知道：

  • start_level=1 说明虽然输入是 4 个特征图，但是实际上 FPN 中仅仅用了后面三个
  • num_outs=5 说明 FPN 模块虽然是接收 3 个特征图，但是输出 5 个特征图
  • add_extra_convs='on_input' 说明额外输出的 2 个特征图的来源是骨架网络输出，而不是 FPN 层本身输出又作为后面层的输入
  • out_channels=256 说明了 5 个输出特征图的通道数都是 256

输出特征图生成流程

  1. 默认输出（P3-P5）：
    • 输入：C3 (stride=8), C4 (stride=16), C5 (stride=32)
    • FPN通过自上而下路径 + 横向连接生成P3, P4, P5（均为256通道）
  2. 额外输出（P6-P7）：
    • 直接对C5（stride=32）进行3x3卷积 + 步长2下采样：
      • P6: C5 -> Conv3x3(stride=2) → stride=64
      • P7: P6 -> ReLU -> Conv3x3(stride=2) → stride=128
    • 关键区别：P6/P7的生成完全独立于FPN的主干路径，不依赖P5的输出

Head

论文中作者认为 one-stage 算法 head 设计比较关键，对最终性能影响较大，相比于其余 one-stage 算法，RetinaNet 的 Head 模块比较重量级。

输出头包括分类和检测两个分支，且每个分支都包括 4 个卷积层，不进行参数共享，分类 Head 输出通道是 num_class*K。

检测 head 输出通道是4*K, K 是 anchor 个数, 虽然每个 Head 的分类和回归分支权重不共享，但是 5 个输出特征图的 Head 模块权重是共享的。
